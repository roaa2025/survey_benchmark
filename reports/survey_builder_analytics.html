Planner Agent

Role:

Analyzes user instructions and survey requirements to create a structured plan for survey question generation

How it works:

Uses AI to:

  • Analyze user instructions (prompt, title, file content, survey type)
  • Extract specific requirements from natural language
  • Determine the target number of questions and pages to generate
  • Create page structure with section briefs
  • Generate writer brief with guidance for question generation

Output:

A structured survey plan (SurveyPlan) with:

  • Page structure (pages with section_brief objects)
  • Question count per page
  • Writer brief (guidance for the Writer agent)
  • Plan rationale and metadata

Writer Agent

Role:

Creates draft survey questions using AI

How it works:

Uses an LLM to generate fully rendered survey questions based on:

  • The Planner's survey plan (title, type, language)
  • Question specifications (question_type, intent, required, options_hint) OR section_brief
  • Writer brief guidance (if provided)
  • User instructions (if provided)

Generates:

  • Question text
  • Options (for radio, checkbox_list, dropdown_list, rank)
  • Scales (for scale, star_rating)
  • Validation rules
  • Skip logic (when logically implied)

Output:

Draft questions in JSON format (may contain errors like missing options, type mismatches, invalid scales)

Critic Agent

Role:

Validates draft questions deterministically

How it works:

Performs deterministic checks on:

  • Spec coverage: all spec_ids appear exactly once, no missing/duplicate/extra spec_ids
  • Question IDs: validates spec_id matches between plan and rendered questions
  • Type matching: question_type matches the spec
  • Required field matching: required flag matches the spec
  • Options validation: radio has ≥2 options, checkbox_list has ≥1, checkbox has exactly 1
  • Scale validation: scale questions have valid min/max integers, star_rating has scale (1-5)
  • Validation rules: email has format="email", required questions have validation.required=true, scale questions have matching min_value/max_value
  • Page names: rendered pages match plan pages

Optionally applies safe auto-fixes when auto_fix=True (e.g., adds missing options, fixes required flags, adds default scales)

Output:

Report containing:

  • passed: boolean (true if no errors)
  • issue_count: number of issues found
  • issues: list of issue objects with severity (error/warning/info), code, message, spec_id, suggestion
  • summary: 2-5 bullet summary of review
  • patched_questions: optionally fixed questions (if auto_fix=True and fixes were applied)

Fixer Agent

Role:

Automatically repairs invalid questions using AI

How it works:

Uses an LLM to fix issues found by the Critic Agent. Applies fixes such as:

  • MISSING_SPEC_ID: adds missing question for the spec_id
  • DUPLICATE_SPEC_ID: removes duplicates, keeps one
  • EXTRA_SPEC_ID: removes questions not in plan (or preserves them during updates)
  • TYPE_MISMATCH: updates question_type to match spec
  • REQUIRED_MISMATCH: updates required field to match spec
  • RADIO_NO_OPTIONS, CHECKBOX_LIST_NO_OPTIONS: adds appropriate options
  • CHECKBOX_OPTION_MISSING: adds exactly 1 option (e.g., "I agree")
  • SCALE_INVALID: fixes scale min/max values and labels
  • STAR_RATING_SCALE_MISSING: adds scale with min=1, max=5
  • EMAIL_FORMAT_MISSING: adds validation.format="email"
  • VALIDATION_REQUIRED_MISSING: adds validation.required=true
  • VALIDATION_SCALE_MISSING: adds validation min_value/max_value matching scale

Preserves all questions without issues (keeps them unchanged)

Output:

Fixed questions in JSON format matching the same schema as input (unfixable questions may be dropped if parsing fails, otherwise original questions are returned)

How Quality Metrics Are Measured

Question Quality (0-100)

  • Clarity: wording is precise, unambiguous, and understandable
  • Answerability: question can be reasonably answered without missing information
  • Non-leading wording: avoids bias, persuasion, or presuppositions

Survey Coherence (0-100)

  • Internal consistency: no contradictions across questions
  • Redundancy control: avoids unnecessary repetition
  • Coverage fit: questions align with survey's user prompt or type
  • Logical grouping: sensible ordering and grouping across pages

Plan Quality (0-100)

  • Plan structure coherence: page organization and logical question distribution
  • Plan-to-question alignment: generated questions match plan specifications
  • Plan metadata accuracy: title, type, and language consistency
  • Spec completeness: percentage of required spec fields properly populated

Question Type Diversity (0-100)

  • Question type distribution: variety across different question types
  • Type appropriateness: each question uses the most appropriate type
  • Type diversity index: number of unique question types used relative to total

Validation Quality (0-100)

  • Spec coverage: all required specs appear exactly once
  • Type matching: question types match their specifications
  • Options validation: selection questions have appropriate options
  • Scale validation: rating questions have proper scale ranges

Controller Appropriateness (0-100)

  • Response Controllers: use of correct and appropriate response controllers (e.g., multiple choice, text input, sliders) for the question being asked

Question-Page Distribution (0-100)

  • Question Distribution: balanced and logical distribution of questions across pages based on user request

Bilingual Alignment (0-100)

  • Semantic equivalence: meaning is preserved across languages
  • Terminology consistency: key concepts are translated consistently
  • Cultural and UX appropriateness: natural phrasing, comparable tone and formality

Score to Label Conversion

Numeric scores (0-100) from the LLM Judge are converted to labels:

  • 90-100: Excellent
  • 75-89: Good
  • 60-74: Fair
  • Below 60: Poor
